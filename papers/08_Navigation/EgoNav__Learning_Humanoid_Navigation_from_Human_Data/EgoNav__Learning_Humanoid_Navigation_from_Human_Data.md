**任务定位**：不收集任何机器人导航数据、也不做微调，**只用人类戴第一视角设备走路采到的轨迹**，学一个**具身无关（embodiment-agnostic）的导航先验**，然后**零样本**部署到人形机器人上做真实环境自主行走。

---

## 🎯 一句话总结

EgoNav 把「导航」拆成一个**与机器人本体无关的轨迹分布预测问题**：用一个 **46M 参数的扩散 UNet**，在仅 **5 小时人类行走数据**上学会「给定过去轨迹 + 360° 第一视角视觉记忆，未来该往哪些方向走」的**多模态轨迹分布**；推理时用 **DDIM+DDPM 混合采样**做到实时（Jetson Thor 上 110 traj/s），再由 **receding-horizon 控制器**从分布里挑路；最终**零样本**迁移到 Unitree G1，在没见过的室内外环境里连续走 **37.5 分钟 / 1137 米 / 96%+ 自主率**，还自发涌现出「等门开、绕人群、避玻璃墙」等行为。

## ❓ 论文要解决什么问题？

人形机器人导航有两个老大难：

1. **机器人数据太贵**：要在真机上采足量的导航演示，既慢又危险，还难覆盖各种场景；
2. **「规划」和「行走」割裂**：传统做法是上层给 waypoint、下层 locomotion 跟踪，中间这层「给定我看到的世界，我下一步该往哪走」往往要为每个机器人单独训。

EgoNav 的核心主张：**「往哪走」这件事本质上和你是人还是机器人无关**——它只取决于「**你过去怎么走的 + 你现在看到了什么**」。于是可以：

- 让**人类**戴第一视角设备在真实世界走路，便宜地采到海量、自然、带避障/社交行为的轨迹；
- 学一个**具身无关的轨迹分布先验**；
- 因为先验不绑定本体，所以可以**零样本**接到人形机器人的 locomotion 控制器上，不需要任何机器人数据或微调。

## 🔧 方法拆解

### 1. 输入：360° 第一视角「视觉记忆」+ 过去轨迹

- **视觉记忆**：一张 `180×360×5` 的 360° 等距全景，**融合 RGB + 深度 + 语义** 五个通道，经一个预训练的 **spatial VAE** 编成 **64 维 embedding**；
- **DINOv3 视频特征**：额外接一个**冻结的 DINOv3** 骨干，抽取**深度传感器看不到的外观线索**（如玻璃墙、地面材质、人群），补足纯几何信息的盲区；
- **过去轨迹**：作为条件，提供运动惯性与意图。

### 2. 模型：扩散 UNet 一次性出「未来轨迹分布」

- **46M 参数的 UNet**，**非自回归**地一次预测**未来 100 步**轨迹；
- 输出是**多模态分布**——同一处路口它会给出「左/右/直走」多条可行路径，而不是单一确定轨迹，这正是导航需要的（避障与社交本就多解）。

### 3. 实时推理：DDIM + DDPM 混合采样

- 直接跑完整扩散去噪太慢。EgoNav 用 **5 步 DDIM（快、确定）+ 5 步 DDPM（保多样性）** 的混合方案，10 步出结果；
- Jetson Thor 上达 **110 条轨迹/秒 @ 1.7 Hz**，并做**延迟补偿**以适配真机闭环。

### 4. 执行：receding-horizon 控制器选路

- 从扩散给出的轨迹**分布**里，按避障/目标对齐挑一条；
- **滚动时域**：只执行前一小段，下一拍用新观测重新预测，保证对动态环境（开门、行人）的反应性；
- 选出的路径作为目标喂给 G1 的底层 locomotion 控制器，**本文不重训 locomotion**。

### 5. 关键数字

| 维度 | 结果 |
|---|---|
| 训练数据 | 仅 **5 小时**人类行走数据，**0** 机器人数据，**0** 微调 |
| 模型规模 | 46M 参数扩散 UNet |
| 实时性 | Jetson Thor 上 110 traj/s，扩散 10 步（5 DDIM + 5 DDPM） |
| 真机部署 | Unitree G1 零样本，**37.5 分钟 / 1137 米 / >96% 自主率** |
| 涌现行为 | 等门开、绕人群、避玻璃墙 |

## 💡 核心贡献

1. **范式**：把导航重新定义为**具身无关的轨迹分布预测**，从而可以用**便宜的人类行走数据**替代昂贵的机器人数据。
2. **零机器人数据 / 零微调**：仅 5 小时人类数据训练，**零样本**迁移到 Unitree G1 真机长时自主行走。
3. **表示**：360° 视觉记忆（RGB+深度+语义）+ 冻结 DINOv3，兼顾几何与外观，能识别**深度传感器盲区**（玻璃墙）。
4. **实时扩散**：DDIM+DDPM 混合采样把扩散轨迹生成压到 10 步、实时可跑，解决扩散策略「慢」的老问题。
5. **涌现社交/交互行为**：等门、绕人群等行为**不是显式编码**，而是从人类数据先验里自然涌现。

## 📊 关键发现

| 维度 | 结论 |
|---|---|
| 数据来源 | 人类行走轨迹足以学到**可迁移的导航先验**，机器人数据不是必需 |
| 多模态性 | 输出轨迹**分布**而非单条路径，是避障/社交导航的关键，受益于扩散建模 |
| 外观线索 | DINOv3 提供的外观语义补上了深度盲区（玻璃/反光），纯几何方案做不到 |
| 实时化 | 混合 DDIM/DDPM 采样让扩散策略在边缘设备（Jetson Thor）上达到可部署频率 |
| 迁移性 | 先验「具身无关」是零样本上机的前提，把规划与具体 locomotion 解耦 |

## 🤖 对人形 / 具身领域的意义

| 方向 | 含义 |
|---|---|
| **人类数据规模化** | 给「用便宜的人类第一视角数据训机器人导航」提供了一个干净、可复制的范式 |
| **规划-行走解耦** | 导航先验只管「往哪走」，locomotion 控制器只管「怎么走」，两者可独立替换/升级 |
| **扩散策略实用化** | 证明扩散轨迹生成可以在边缘算力上实时跑，推动扩散从仿真走向真机 |
| **与 H\* / NaVILA / FocusNav 的位置** | 那些工作偏「看哪里 / 语义规划 / 局部注意力」；EgoNav 给的是**「下一步轨迹分布」**这一中间层，可作它们的执行落地 |

## 🎤 面试参考

**Q：为什么说导航先验可以是「具身无关」的？人和机器人腿都不一样啊。**
A：EgoNav 学的不是「怎么迈腿」，而是「**给定我过去怎么走 + 我现在看到什么，我接下来该往哪些方向走**」——这是一个**轨迹层面**的决策，跟你用两条人腿还是 G1 的关节没关系。迈腿那一层交给机器人自己的 locomotion 控制器。把「往哪走」和「怎么走」解耦，前者就能跨本体复用，于是人类数据可直接迁移。

**Q：为什么要输出轨迹「分布」而不是一条最优路径？**
A：真实导航本就多解：路口可左可右、绕人可从左绕也可从右绕。如果模型只学单条「平均路径」，在分叉处会塌成一条不可行的中间路（mode averaging）。扩散模型天然建模多模态分布，再由 receding-horizon 控制器按避障/目标实时挑一条，既保多样性又保可行性。

**Q：DINOv3 在这里到底补了什么纯深度做不到的事？**
A：补**外观语义**。玻璃墙、反光地面、人群这些，深度传感器要么穿透看不到、要么噪声大；而冻结的 DINOv3 从 RGB 抽出的特征能区分「这是可穿过的开口还是一堵玻璃」「前面是人还是柱子」。几何 + 外观一起喂，才有「避玻璃墙、绕人群」这种涌现行为。

**Q：扩散模型一般很慢，EgoNav 怎么做到实时？**
A：两点。其一，**非自回归**一次出 100 步轨迹，不像自回归逐步生成；其二，**混合采样**——5 步 DDIM（确定、快）打底，再 5 步 DDPM（随机、保多样性）补分布，总共 10 步而不是几十上百步去噪。配合延迟补偿，Jetson Thor 上 110 traj/s，足够闭环。

**Q：完全零机器人数据，迁移时最大的风险是什么？**
A：**人类与机器人在尺寸、视高、动力学上的差异**。人走得比 G1 灵活、视角高度也不同，先验里某些「人类能做、机器人做不动」的轨迹会失败。EgoNav 靠 receding-horizon 不断重规划 + 底层 locomotion 兜底来吸收这部分 gap，真机上拿到 96%+ 自主率，说明轨迹层先验对本体差异有相当的鲁棒性，但这也是它后续要补真机数据微调的地方。
