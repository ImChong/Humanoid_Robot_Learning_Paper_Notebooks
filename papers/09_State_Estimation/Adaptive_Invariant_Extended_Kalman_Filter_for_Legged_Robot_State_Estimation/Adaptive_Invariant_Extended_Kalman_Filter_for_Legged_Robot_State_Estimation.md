## 🎯 一句话总结

经典 **Contact-Aided InEKF**（不变扩展卡尔曼滤波）把"支撑足不动"当成一条理想测量，用一个**固定的接触噪声协方差**去信任它；可一旦足底发生**小幅打滑**，这个假设就被破坏、状态估计随之漂移。本文给 InEKF 加上一套**在线协方差自适应**机制：根据实时残差估计接触足模型的真实噪声水平，**打滑时自动放大该足的测量噪声、站稳时再收回**，从而在不增加任何接触力传感器（仅靠接触检测算法）的前提下，把传统"硬阈值打滑剔除"处理不了的小滑动也吸收掉，在 LeoQuad 的动态运动中拿到更稳的里程。

## ❓ 论文要解决什么问题？

足式机器人的本体感知状态估计普遍建立在 **Contact-Aided InEKF** 上，它的核心假设是：**当前支撑足在世界坐标系中保持静止**，由此给 base 速度提供一条强约束。问题在于：

1. **接触噪声协方差是手调的固定值**——它隐含假设"支撑足真的不动"。
2. 现实里足底经常**小幅打滑**（软地、斜面、快步态、触地瞬间），这条"不动"约束被悄悄违反；
3. 传统补救是**打滑剔除（slip rejection）**：用一个硬阈值判断"打滑就丢掉这只脚的约束"。但**小滑动**往往达不到阈值、又确实污染估计，成了夹缝里的漏网之鱼；
4. 若想精确判断接触 / 打滑，常要加**接触力传感器**，增加硬件成本与故障点。

本文要回答的就是：**能不能不加传感器、也不靠硬阈值，让滤波器自己根据数据判断"这只脚现在有多可信"？**

## 🔧 方法拆解

### 1. 底座：Contact-Aided RIEKF

- 状态（base 位姿、速度、IMU 零偏、各接触足位置）定义在矩阵李群上，IMU 做**过程模型**（预测步）；
- 每个被判定为支撑的足，提供一条"足在世界系静止 / 前向运动学一致"的**测量更新**；
- 不变误差动力学与状态轨迹无关，带来比标准 EKF 更好的收敛与一致性。

### 2. 关键改动：接触足噪声的「在线自适应」

把固定的接触测量协方差 **R** 换成**随时间在线估计的 R̂_k**：

- 用滤波器的**新息 / 残差序列**（实际测量与预测之差）做在线协方差估计；
- 当某只支撑足**频繁打滑**时，它的残差统计会变大 → 自适应机制**调大该足的 R̂** → 滤波器**少信任**这条约束；
- 当足底**站得很稳**时，残差小 → **R̂ 收回**到小值 → 滤波器**多信任**它、估计更紧。

> 直觉：把"信不信这只脚"从**离散开关**（打滑剔除）升级成**连续旋钮**（协方差自适应），小滑动也能被平滑地"减权"而不是粗暴丢弃。

### 3. 用「接触检测算法」替代接触力传感器

- 不依赖足底力 / 触觉传感器，而是用**算法**（基于运动学 / 动力学 / 力矩信号）判断哪只脚处于支撑相；
- 降低硬件依赖与成本，同时把"接触判定"和"接触可信度"两件事解耦：前者用检测算法给出，后者交给协方差自适应连续刻画。

### 4. 整体闭环

IMU 高频预测 → 接触检测选出支撑足 → 该足给出运动学测量 → **自适应模块**据残差更新各足协方差 → InEKF 用更新后的协方差做校正 → 输出 base 位姿 / 速度，并把残差喂回自适应模块，形成闭环。

## 💡 核心贡献

1. **接触噪声协方差从「固定手调」升级为「在线自适应」**：让 InEKF 自己根据残差判断每只支撑足当前有多可信。
2. **专治小幅打滑**：传统硬阈值打滑剔除够不着的小滑动，被自适应协方差平滑减权，避免污染状态。
3. **无需接触力传感器**：用接触检测算法替代触觉硬件，降低成本与故障点。
4. **真机验证**：在四足机器人 LeoQuad 的动态运动场景下验证，状态估计精度优于固定协方差的基线 InEKF。

## 📊 关键发现

| 维度 | 结论 |
|---|---|
| 打滑鲁棒性 | 在频繁小幅打滑场景下，自适应版本相比固定协方差 InEKF 漂移更小 |
| 硬件需求 | 仅本体感知（IMU + 关节 + 接触检测），**无额外接触力传感器** |
| 适用条件 | 动态步态、软 / 滑地面等"支撑足并非严格静止"的工况收益最明显 |

## 🤖 对人形 / 状态估计领域的意义

| 方向 | 含义 |
|---|---|
| **直接可移植到人形** | 人形支撑面更小、姿态更高，足底打滑对估计的污染比四足更严重，"连续减权"思路对双足尤其有价值 |
| **少一类传感器依赖** | 不依赖足底力 / 触觉，对量产硬件与成本敏感的人形平台友好 |
| **与学习式里程计互补** | 与 AutoOdom 这类纯学习路线相比，本文坚持解析 InEKF 但补上自适应能力，是"模型驱动"阵营对打滑问题的正面回应 |
| **后端可叠加** | 输出标准的 base 位姿 / 速度，可作为因子图 / SLAM 后端（GTSAM 等）的前端里程来源 |

## 🎤 面试参考

**Q：Contact-Aided InEKF 为什么会因为打滑而漂移？**
A：它的测量更新建立在"支撑足在世界系中静止"这条约束上，并用一个**固定的小协方差**表示"我很信任这条约束"。一旦足底打滑，约束本身被违反，但协方差还是那个小值——滤波器仍然过度信任一条错误测量，状态自然被往错误方向拉，累积成漂移。

**Q：自适应协方差和传统的打滑剔除有什么本质区别？**
A：打滑剔除是**离散硬开关**——超过阈值就把整只脚的约束丢掉，够不着阈值的小滑动则原样污染估计。自适应协方差是**连续旋钮**——根据残差统计实时调大 / 调小该足的测量噪声，小滑动也能被平滑减权，既不会"一刀切丢信息"，也不会"全盘照收坏信息"。

**Q：不用接触力传感器，怎么知道哪只脚在支撑？**
A：用**接触检测算法**，基于运动学 / 关节力矩 / 动力学一致性等本体信号推断支撑相。本文把"是否接触"（检测算法）与"接触有多可信"（协方差自适应）解耦：前者决定要不要用这条约束，后者决定用多大权重。

**Q：这套方法迁到人形机器人上会更难还是更容易？**
A：动机更强、也更需要。人形是双足、支撑面小、重心高，足底微滑对 base 速度估计的影响被放大；固定协方差的 InEKF 在双足上更容易被打滑带偏，所以"按打滑程度连续减权"的收益在人形上通常比四足更明显——前提是接触检测在双足相位切换频繁的情形下仍要够准。
