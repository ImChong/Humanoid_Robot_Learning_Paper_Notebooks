## 🎯 一句话总结

腿部里程计的命门在于**判断「支撑相」**——脚一旦落地静止，就能用**零速更新（ZUPT）**给状态估计一条强约束。传统做法依赖**足底力 / 地面反作用力（GRF）传感器**判断接触，但这类传感器贵、在**打滑和软地形**下还经常失效。本文提出一个**自监督接触表征学习**框架：用**去噪自编码器（DAE）**把仅靠关节编码器算出的运动学序列（足高、足速、小腿力矩）投影到一个低维潜空间，**支撑相与摆动相在潜空间里自然分开成两簇**；再用**高斯混合模型（GMM）**自动给两簇贴上「支撑/摆动」标签，输出的是**连续的支撑概率**而非硬 0/1。这个概率直接拿去**动态缩放 ESEKF 的测量协方差**——越不像支撑，协方差越大、滤波器越不信任这只脚——从而在**不加任何力传感器、不需要人工标注**的前提下拿到更稳的腿部里程计。

## ❓ 论文要解决什么问题？

足式里程计的核心逻辑是：**支撑相时脚不动 → 用 ZUPT 约束 base 速度**。要用好它，必须先**准确判断哪只脚在支撑**。现有路线有两类痛点：

1. **依赖 GRF / 力传感器**：硬件贵、易损，且在**打滑、地形顺应（软地）**时力信号本身就不可靠，接触判定跟着失准；
2. **监督学习接触分类器**：需要**人工标注**支撑/摆动标签，标注成本高、还容易过拟合到训练地形；
3. 更隐蔽的一点：**分类指标好 ≠ 里程计准**——一个 F1 很高的接触分类器，未必能给下游滤波器带来最好的轨迹精度（本文实验恰恰证明了这点）。

本文要回答：**能不能只用关节编码器、不要力传感器、也不要人工标注，自监督地学出一个「接触表征」，并且让它真正服务于下游里程计精度？**

## 🔧 方法拆解

### 1. 输入：只要关节编码器能算出来的 5 维特征

每只脚构造一个 **5 维特征向量**：**足底垂直位置（1）+ 足速度（3）+ 小腿力矩（1）**，全部由关节编码器 + 运动学/动力学算出，**不含任何 GRF 信号**。

### 2. 去噪自编码器（DAE）：把运动学投影到「会分簇」的潜空间

- 对输入做**污染（corruption）**，让 **CNN 或 GRU 编码器**（潜维 **D=16**）+ 转置卷积解码器去**重构干净序列**，损失是 MSE；
- 关键效果：在被逼出的低维潜空间里，**支撑相和摆动相的样本自然落到两个分离的簇**，无需任何标签。

### 3. GMM 自动贴标签 → 输出连续支撑概率

- 在潜空间拟合**高斯混合模型**，得到两个簇；
- 用启发式规则定支撑簇：**「垂直足位更低、速度方差更小」的那簇判为支撑**；
- 输出**连续的支撑概率 pₜ(stance)**，而不是硬 0/1。

### 4. 把置信度灌进 ESEKF：用概率缩放测量协方差

ZUPT 测量协方差按支撑概率动态调节：

> **Rₜ = ( 1 / pₜ(stance) + ε ) · I₃ₓ₃**

支撑置信度越低 → 协方差越大 → 滤波器**越不信任**这只脚的零速约束，自动把打滑 / 误触地的坏测量「软性减权」掉，而不是一刀切丢弃。

### 5. 一个反直觉的设计选择：窗口 W=1 最好

消融显示**单帧窗口（W=1）表现最佳**；大窗口（如 W=100）会把**周期步态卷成一个环状流形（ring manifold）**，破坏 GMM 的高斯假设、反而变差。潜维 D=16（D<4 压缩过度、D>4 收益递减）。所有方法单步 <10ms，DAE 网络仅 **3.0ms**，可实时。

## 💡 核心贡献

1. **自监督接触表征**：用 DAE + GMM，仅靠关节编码器、无力传感器、无人工标注，就把支撑/摆动分出来。
2. **输出连续置信度而非硬接触**：支撑概率直接缩放 ESEKF 协方差，对打滑 / 误触地做平滑减权。
3. **指出「分类指标 ≠ 里程计精度」**：监督方法 F1 更高，但 DAE 路线在下游轨迹误差上反而更优。
4. **真机 + 仿真双验证 + 开源**：ANYmal（仿真）与 Unitree Go2（真机）上均验证，代码公开。

## 📊 关键发现

| 指标 | 结果 |
|---|---|
| 接触分类 F1（真机 Go2） | DAE-GRU 0.900 / HMM-GMM 0.906 vs 监督 GRU 0.921——**自监督略低但接近** |
| 里程计 ATE（仿真 ANYmal） | **DAE-CNN 4.09m（最佳）** < GRF 阈值 4.25m < 监督 CNN 4.32m |
| 里程计 ATE（真机 Go2，1.10km） | **DAE-CNN 17.20m（最佳）** < 监督 CNN 21.95m < HMM-GMM 27.13m |
| 航向误差 AHE（真机） | **DAE-CNN 31.90° < 监督 CNN 44.67°** |
| 实时性 | DAE 推理 3.0ms / 步，所有方法 <10ms |

> 📌 数据来自论文表 1–3；真机用 RTK GPS 作真值，覆盖混凝土 / 草地 / 碎石。

## 🎤 面试参考

**Q：为什么腿部里程计这么依赖「接触检测」？**
A：因为支撑相时支撑足在世界系里几乎静止，可以用零速更新（ZUPT）给 base 速度一条强约束，抑制 IMU 积分漂移。判错支撑相 → 要么漏掉好约束、要么把打滑的脚当静止用，都会把里程带偏。

**Q：去噪自编码器为什么能无监督地把支撑/摆动分开？**
A：DAE 被逼着从被污染的运动学里重构干净信号，会把数据里**最稳定、最有结构的因素**压进低维潜空间。对腿部运动学来说，「脚在地上不动」和「脚在空中摆动」是两种截然不同的动力学模式，于是它们在潜空间自然落成两簇，再用 GMM 一聚类就分出来了。

**Q：论文说「分类指标好不等于里程计准」，怎么理解？**
A：监督分类器 F1 更高，但它优化的是「逐帧标签对不对」，而里程计在意的是「触地/离地**时刻**判得准不准、对坏测量减权得够不够」。DAE 输出连续概率、对边界样本更平滑，下游 ESEKF 因此拿到更稳的协方差调度，轨迹误差反而更小——这说明评估接触模块要看**端到端里程精度**，不能只看分类分。

**Q：为什么单帧窗口 W=1 比长窗口好？**
A：长窗口会把周期性步态卷成一个环状流形（ring manifold），潜空间分布不再是两个高斯团而是一个环，破坏了 GMM 的高斯假设，聚类质量下降；单帧反而让支撑/摆动两态在潜空间最干净地分开。
