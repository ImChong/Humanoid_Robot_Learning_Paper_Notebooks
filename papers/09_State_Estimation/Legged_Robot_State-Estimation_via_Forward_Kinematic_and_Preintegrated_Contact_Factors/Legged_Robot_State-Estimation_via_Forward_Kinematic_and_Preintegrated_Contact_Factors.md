# Legged Robot State-Estimation Through Combined Forward Kinematic and Preintegrated Contact Factors
**把 IMU 预积分的思路搬到「腿」上：用前向运动学因子约束 base↔足端、用预积分接触因子把高频接触测量压成一条相对位姿约束，整体在因子图上做平滑（smoothing）而不是滤波（filtering）**

## 🎯 一句话总结

这篇文章把视觉惯性里程计里大获成功的 **IMU 预积分（preintegration）** 思想推广到「腿」上：它引入两类新因子——**前向运动学因子**（用关节编码器把 base 位姿和触地足端位姿联系起来）和**预积分接触因子**（把支撑相内高频的接触/运动学测量压缩成关键帧之间的一条相对位姿约束，并让噪声随滑移随时间增长）——再与标准 IMU 预积分因子一起塞进**因子图**做增量平滑（iSAM2 / GTSAM），从而用 smoothing 取代传统的 EKF filtering，得到更准、更易扩展（可随手挂上视觉 / 回环等额外因子）的足式浮动基状态估计。

## ❓ 论文要解决什么问题？

足式机器人在跑控制器之前要估计**浮动基状态**（base 的位姿、速度）。主流做法是 **EKF/InEKF 滤波**：用 IMU 做预测，用「触地脚是世界系里固定点」这个运动学约束做更新。但滤波路线有几个痛点：

1. **只递推当前状态、无法重线性化**：早期线性化误差会一直累积，无法回头修正；
2. **多传感器异步融合笨重**：要把视觉 / LIDAR / 回环这类延迟、异步的测量塞进 EKF 很别扭；
3. **接触测量频率极高**：每个高频接触/运动学测量都当一次更新，计算与建模都不优雅。

本文主张换条路——**用因子图平滑代替滤波**，并解决随之而来的核心难题：*怎样把高频的腿部接触信息高效地表达成因子，而不至于让图爆炸？* 答案是借鉴 IMU 预积分，发明「预积分接触因子」。

## 🔧 方法拆解

### 1. 总体框架：一切都是因子图里的因子

状态节点取在一串关键帧上（base 位姿 / 速度 / IMU 偏置，以及触地足端的位姿）。三类因子把它们连起来：

| 因子 | 连接对象 | 来源 | 作用 |
|---|---|---|---|
| **IMU 预积分因子** | 相邻 base 关键帧 | 加速度计 + 陀螺仪 | 给出两帧间的相对运动（沿用 Forster et al. 的经典预积分） |
| **前向运动学因子** | base 位姿 ↔ 同时刻触地足端位姿 | 关节编码器 + FK | 用腿的几何把 base 和「脚」绑在一起，协方差由编码器噪声经 FK 雅可比传播 |
| **预积分接触因子** | 相邻触地足端关键帧 | 接触/运动学高频测量 | 在支撑相内，触地足端在世界系里「本应不动」；本因子预积分其相对位姿，并让协方差随滑移随时间增长 |

### 2. 关键创新一：前向运动学因子（Forward Kinematic Factor）

关节编码器给出关节角 α，前向运动学 h(α) 给出**足端相对 base 的位姿**。当某只脚触地时，这条 FK 测量把 `base 位姿` 与 `该足端在世界系的位姿` 直接关联。关键细节是**协方差传播**：编码器噪声经 FK 的解析雅可比映射成足端位姿的不确定度——脚伸得越远 / 构型越奇异，这条约束就越「松」，这点被显式建模进因子里。

### 3. 关键创新二：预积分接触因子（Preintegrated Contact Factor）

这是全文的灵魂。直接为每个高频接触测量建一个变量/因子会让图爆炸。借鉴 IMU 预积分：

- 在一只脚的整段支撑相里，**理想刚性接触下足端世界位姿不变**，但真实接触会**滑移 / 微转**；
- 把这段时间内触地足端的**相对位姿增量**预积分成一条因子，连接支撑相首尾两个足端关键帧；
- **噪声模型**：滑移被建成随机游走，协方差**随接触时长增长**——脚踩得越久、越可能滑，约束就越不可信。这让滤波器/平滑器对「长时间支撑后的微小滑移」保持恰当的怀疑。

> 直觉：IMU 预积分把「两帧之间一堆加速度计读数」压成一条相对位姿约束；预积分接触因子把「支撑相内一堆接触读数」同样压成一条相对位姿约束。两者结构完全对偶。

### 4. 为什么要 smoothing 而不是 filtering？

- **可重线性化**：因子图在每次增量求解时能回头重线性化早期因子，抑制累积线性化误差；
- **天然多传感器 / 异步 / 延迟友好**：视觉、LIDAR、回环只是「再挂一个因子」，不需要改滤波器结构；
- **增量高效**：用 iSAM2 只更新受影响的子图，能逼近实时。

## 💡 核心贡献

1. **预积分接触因子**：首次把 IMU 预积分的「压缩高频测量」范式迁移到腿部接触上，让支撑相内成百上千的接触测量浓缩为一条带「滑移随时间增长」噪声模型的相对位姿因子，避免因子图爆炸。
2. **前向运动学因子**：把关节编码器 + FK 显式建成约束 base↔足端的因子，并严谨地用 FK 雅可比传播编码器协方差，使「腿越伸越不可信」这件事被自动量化。
3. **从滤波走向平滑**：用因子图（GTSAM / iSAM2）替代 EKF/InEKF，获得可重线性化、可异步/延迟融合、可随手挂额外传感器的统一框架。
4. **与同组 InEKF 工作互为镜像**：本文（2018, smoothing）与同组次年的 Contact-Aided InEKF（2019, filtering，本仓库已有笔记 1904.09251）共享同一套「接触辅助 + 前向运动学」物理直觉，分别代表平滑 / 滤波两条技术路线。

## 📊 关键发现（基于论文结论）

| 维度 | 结论 |
|---|---|
| **精度** | 在 Cassie 系列双足数据上，平滑方案相比经典 EKF 基线降低了浮动基估计漂移 |
| **接触因子价值** | 预积分接触因子让「长支撑相滑移」被正确地建成增长的不确定度，避免对滑移过度自信 |
| **可扩展性** | 因子图框架下加视觉 / 回环只是「再挂因子」，无需重构估计器 |
| **实时性** | 借助 iSAM2 增量求解，仅更新受影响子图，可逼近在线运行 |

## 🎤 面试参考

**Q：预积分接触因子到底「预积分」了什么？**
A：预积分了一只脚在整段支撑相内、其触地点在世界系中的相对位姿增量。理想刚性接触下该增量应为零（脚不动），但真实接触有滑移/微转，于是把滑移建成随机游走，协方差随接触时长增长。这样支撑相内成百上千的高频接触测量被压成关键帧之间的一条相对位姿因子——结构上和 IMU 预积分把一堆加速度计读数压成一条相对运动因子完全对偶。

**Q：前向运动学因子和「触地脚是世界系固定点」的 EKF 更新有何关系？**
A：物理直觉一样（都用关节编码器 + FK 把 base 和脚联系起来），但表达形式不同。EKF 把它当作一次观测更新、只影响当前状态；本文把它表达成因子图里连接 base 关键帧和足端关键帧的一条约束，可被反复重线性化、可与其它因子联合优化。关键工程点是用 FK 雅可比把编码器噪声正确传播成足端位姿协方差。

**Q：既然 smoothing 更准，为什么很多机器人仍用 InEKF 滤波？**
A：算力与延迟。滤波只递推当前状态、计算恒定、延迟极低，适合做控制器的高频反馈（kHz 级）；因子图平滑要维护一段历史、求解更重，即便 iSAM2 增量化，时延和实现复杂度仍更高。实践中常见做法是：底层用 InEKF 给控制器喂高频估计，上层用因子图平滑做更准的全局轨迹/地图。

**Q：这套方法怎么扩展到加视觉或回环？**
A：因为是因子图，加视觉只是再往图里挂视觉里程计/特征重投影因子，加回环就挂一条回环约束因子，估计器主体完全不用改——这正是 smoothing 相对 filtering 的核心工程优势。
